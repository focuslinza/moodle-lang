Moodle 1.5 (5. Juni 2005)

Zenrale Features

  • Webseiten sind jetzt XHTML Transitional 1.0
  • Improved Accessibility, in Übereinstimmung mit WAI (W3C), SENDA (UK) und Section 508 (US) Kriterien.
  • Neues Themes system, mit Cascading Themes, Nutzerthemes, Kursthemes, Seitenkontrolle über CSS.
  • Integriertes Messaging Feature zur direkten Kommunikation zwischen Nutzer/innen, Pop-Up-Fenster mit neuen Nachrichten, Emailbenachrichtigung, Sperren von Nachrichten, Archiv, WYSIWYG-Editor etc
  • Forum gekesen/ungelesen Tracking mit Hervorhebungen macht neue Beiträge auf einen Blick sichtbar. Einstellung auf Forums, Nutzer oder Seitenebene.
  • Neues Blocksystem, erlaubt mehrere Blockkopien und bessere Konfiguration der Blöcke
  • Komplette Unterstützung der Sommerzeiteinstellung weltweit.
  • Neues erweitertes Notenbuch ermöglicht die Gewichtung verschiedener Aktivitäten, die Vergabe von Extrabewertungen oder die Herausnahme anderer und weiteree Anzeige und Sortieroptionen
  • Nutzer können aufgefordert werden Policyerklärungen zu bestätigen

Andere systemweite Änderungen

Admin

  • Neu "Wartungsmodus" ermöglicht zeitweise Abschaltung der Seite (z.B. während eines Upgrades).
  • Zeitzonen können direkt von moodle herunter geladen und in der Datenbank installiert werden (Olson werden ebenfalls unterstützt)
  • Neue Version des Moodle MySQL Admin Moduls (separat verfügbar)
  • Neue Kalendaroptionen erweiterte Einstelloptionen

Authentifizierung

  • Neu Shibboleth Support
  • Neu CAS Support
  • Neu PAM Support
  • LDAP Integration verbessert, LDAP kann nun die Kurserstellung kontrollieren, Gruppen zuordnen. Gesteigerte Leistzung und Skalierung der Leistung bei der Nutzersynchronisation.
  • Einzelne Nutzerfelder können vom Admin gesperrt werden wenn die externe Nutzerauthentifizierung verwendet wird
  • Verbessertes session handling erkennt und behebt jetzt "crossover" Sessions, die in einigen fehlerhaften PHP Installationen festgestellt wurden
  • Sessions können nun in der Datenbank gespeichert werden (für clustered Installationen!)
  • Nutzer/innen können gezwungen werden, ihr Passwort zu ändern

Dateien

  • Standardisiertes Suchen von Dateien in allen Bereichen
  • Verbesserter Upload, support für automatische Virenerkennung bei neuen Dokumenten mit ClamAV.
  • Slasharguments arbeitet nun auch unter IIS (upgrade nach PHP 4.3.11 erfordert ISAPI)

Filter

  • Standardisierte Filterbibliothek erleichtert die Erstellung neuer Filter für die Suche und Markierung von Text
  • Wesentlich verbesserte Effizienz bei komplexen Filterungen wie beim Glossarfilter
  • Neuer Tidyfilter nutzt das W3C Tidy Programm um alle Texte zu prüfen und mit gültigem XHTML-Code auszuliefern
  • Verbesserter Wortzensurfilter, markiert zensierte Wörter mit einem schwarzen Balken und erlaubt die Erstellung von Wortlisten für jede Sprache
  • Verbesserter Flash MP3 Player, zeigt Fortschrittsanzeige beim Download und Abspielen
  • Verbesserter Multi-language-Filter ist schneller, toleranter bei Syntax Fehlern und verwendet eine neue einfachere Syntax: <span lang="en"></span>
  • Auto-linking Filter (glossary, activities) erkennt alle vorkommenden Begriffe (altes Verhalten), einmal pro Textblock oder einmal pro Seite ($CFG->filtermatchonepertext, $CFG->filtermatchoneperpage).
  • Weitere Textelemente können gefiltert werden, Namen von Lernaktivitäten, Überschriften und andere kleinere Textteile. Damit können vollständig mehrsprachige Seiten entwickelt werden, die in der Nutzersprache angezeigt werden.

Themes

  • Module können Standard Styles nutzen
  • Themes können Styles anderer Themes überschreiben
  • Nutzer und Kurse können eigene Themes wählen
  • Implementierung von Seiten mit Tabs
  • Module, Blöcke und Sprachen können Standardstyles erhalten

HTML Editor

  • Die Toolbuttons desEditors können vom Admin konfiguriert werden
  • Suchen und Ersetzen-Feld im Editor

Kursmanagement

  • Transparenter Blackboard 5.5 Kursimport (teilweiser 6.0 Support)
  • Neue MetaKurse erlauben die Integration von Teilnehmernn aus anderen Kursen, die mit dem Metakurs verbunden werden
  • Neues Tool zum Kopieren von Kursen von einem Kurs zu einem anderen.

Neue Blöcke

  • HTML Block: ermöglicht beliebigen Inhalt einzufügen (Text, Bilder, Links) auf jeder Kursstartseite
  • Remote RSS Feed: ermöglicht die Anzeige anderer RSS Channels in einem Moodlekurs
  • Glossar Selection: auswählen und anzeigen von Inhalten aus einem Glossar auf der Kursstartseite

Kalendar

  • Deutliche Geschwindigkeitsverbesserung bei Seiten mit vielen Kursen und Gruppen
  • Sich wiederholernde Termine können nun zusammen oder einzeln geändert und gelöscht werden
  • Neue "remember filter settings" erlaubt Kalenderfilterung eine Erinnerung an den Status zwiscehn Loginspreference that allows calendar filters to remember their status between logins

Lernaktivitäten

Aufgabet

  • Vollständig überarbeitet zu einen klassenbasierten Tool, das das Plugin neuer Aufgabentypen erlaubt
  • Neuer Online-Text-Aufgabentyp, der Korrektur in einer Textkopie und Beurteilung erlaubt. Dieser Auzfgabentyp ersetzt das bisherige Journalmodul
  • Erheblioch verbessertes Beurteilungstool, besonders für große Klassen
  • Neue Konfigurationsoptionen, um verspätete Abgaben zu unterbinden und Trainer/innen per Email überneue abgegebene Lösungen zu benachrichtigen

Chat

  • Oberfläche verbessert, arbeitet schneller und auch ohne den optionalen Server Daemon
  • Blöcke könenn zu Chatseiten hinzugefügt werden (z.B. für zusätzliche Informationen)

Abstimmung

  • Überarbeitet zur Verwendung jeder Zahl von Abstimmungen
  • Die Zahl der Nutzer/inenn je Alternative kann begrenzt werden (z.B. für die Bildung von Gruppen mit Höchstteilnehmerzahl.
  • Download der Ergebnisse als XLS oder TXT-Datei

Forum

  • Neues Google-ähnliches Forensuchtool
  • Forum gelesen/ungelesen tracking - ungelesene Mitteilungen werden hervorgehoben auf der Kursseite, der Themenübersicht und in der Beitragsanzeige
  • Das Trackingsystem kann von trainer/innen / Teilnehmer/innen abgeschaltet werden
  • Nutzerprofile zeigen alle Beiträge nach Nutzern
  • Wenn Admins Beiträge überarbeiten wird eine Nachricht hinterlassen
  • Diskussionslisten zeigen den letzzten User, der zu einem Thema gepostet hat, wann dies erfolgt und einen Link zu diesem Beitrag
  • In Gruppenforen wird die zugehörige Gruppe und ein Link zur Gruppenübersicht gezeigt.

Glossar

  • Neue Einstellung zum (de-)aktivieren der Druckfunktion.
  • Neue Suchfunktion sucht auch Einzelwörter statt nur Phrases.
  • Mehr Informationen in den Logs.

Journal

  • Das Journalmosul wurde seit Version 1.4 nicht weiter entwickelt und wird nun deaktiviert. Beim Upgrade werden alle Journalbeiträge in OnlineAufgaben überführt. Alte Journalaktivitäten werden ausgeblendet. Um dies zu verhindern können Sie in der config.php folgende Einstellung einfügen: $CFG->noconvertjournals = true;

Lektion

  • unterstützt jetzt zeitlich begrenzte Lektionen
  • Kann nun Übungslektionen erstellen (Noten werden nicht gespeichert)
  • Option für Teilnehmer/innen Punkte, die bereits erreicht wurden während der Lektion zu sehen
  • Erlaubt Teilnehmer/innen Antworten einzusehen, bevor Ergebnisse abgespeichert werden
  • Neuer Slide Show Modus (nur Branchtables werden als Slides gezeigt)
  • Neues Link Menu (nur für Branchtables)
  • Lektionen können passwortgesichert werden
  • Baumübersicht kann bei der Erstellung von Lektionen verwandt werden
  • TN können Höchstpunktzahlen posten
  • Neue Option zum Speichern der Lektioneneinstellungen als Grundlage für weitere Lektionen im gleichen Kurs
  • Versuche der TN können gelöscht werden
  • Neue Seite Sprünge:
    • Vorherige Seite
    • Ungesehene Fragen eines Zweiges
    • Zufallszuordnung der Fragen
    • Ungesehene Fragen eines Clusters
  • Zusätzliche Funktion zu Lektionststatistiken
  • Neuer Fragentyp: Essay. Trainer können jeden Text einsehen, kommentieren, bewerten und an Tn per Email senden
  • Neues Interface zur Fragenerstellung

Test

  • Erlaubt angepasste Fragen, z.B., Fragen, die sich an das Verhalten des TN anpassen und je nach Antwort verändert werden.
  • THn kann erlaubt werden eine Frage immer wieder zu bearbeiten bis die Antwort richtig ist.
  • Ein Abzugsmechanismus kann den Punktwert für die Antwort reduzieren bei mehrfacher Bearbeitung einer Frage.
  • Ermöglicht die nahtlose Verwendung und Bewertung von extertnen Fragen (z.B., von Mathematischen Aufgabenmaschinen) über das RQP web services protocol.
  • Vorbereitung für künftige IMS QTI Fragen durch web services .
  • Trainer Interface mit Tabs für Vorschau, Bearbeitung und Einsichtin durchgeführte Tests
  • Neuer "security" Modus zeigt Tests im neuen Full-Screen-Modus, mit deaktivierten Browserfunktionen
  • Tests können über mehrere Seiten verteilt werden. Die Zahl der Fragen pro Seite kann festgelegt werden.
  • Blöcke können auf Testseiten eingefügt werden (für Ergebnisübersichten oder ergänzende Informationen)
  • Bessere Oberfläche zur Fragenauswahl und Fragenmanagement in der Datenbank. Ausgewählte Fragen könenn dem Test hinzugefügt, in Kategorien verschoben oder gelöscht werden.
  • Zufallsfragen werden zur Fragenliste über einen Button hinzugefügt. Besseres Handling
  • Endnote kann Dezimalzahl sein
  • Verbesserte Ergebnisübersicht mit gestaltbarer Anzeige der Tabellen, besserer Sortierung und neuer Anzeige
  • Fragen können mit einem Klick aus der Vorschau übernommen werden.
  • Fragen könenn mit einem Klick kopiert werden, um Fragen einfacher zu überarbeiten
  • Überarbeitete Struktur für Fragenkategorien und -bearbeitung.
  • Kategorien können verschachtelt und umorganisiert werden.
  • Auswahl ob Fragen aus Kategorien oder Unterkategorien im Test gezeigt werden sollen.
  • Zufallsfragenauswahl aus einer Kategorie oder aus der Kategorie und ihren Unterkategorien.
  • Testbearbeiter können Fragen nicht bearbeiten bei denen Antworten von Tn vorhanden sind.
  • Erkennung und Verhinderung von überlappenden und redundanten Fragen wurde erstellt.
  • Anzeige von Fragen in Kategorien mit sehr vielen Fragen über mehrere Seiten.
  • Neue Exporttypen - IMS QTI (2.0) und xhtml. Zusätzlich zu anderen
  • Neuer Importtyp - Hot Potato. Entwicklung und Überarbeitung anderer Formen.

Scorm

  • SCORM 1.2 standardkompatibel.
  • Nutzerergebnisse werden gespeichert
  • AICC Pakete können importiert werden.

Umfrage

  • Grafische Darstellung der Ergebnisse

Wiki

  • Änderungen auf Wikiseiten werden ausgewertet und auf der Kursseite unter Letzte Aktivitäten angezeigt
  • Verbesserungen bei Backup/Restore
