Text formatieren

Wenn Sie in Moodle einen Text schreiben, können Sie zwischen mehreren Formaten wählen. Die Wahl hängt von Ihrer Fachkenntnis und dem Browsertyp ab, den Sie verwenden.

Im Normalfall können Sie die Einstellungen so belassen wie sie sind.

1. Moodle-Text-Format

    Dieses Format verwendet man am besten, wenn man normale Web-Formblätter für Eintragungen benutzt (anstatt den Rich-Text-Editor). Dazu tippen Sie den Text einfach so, als würden Sie eine normale E-Mail schreiben.

    Wenn Sie Ihren Text speichern, wird Moodle in den meisten Fällen Ihren Text automatisch formatieren.

    URLs wie: http://yahoo.com oder www.yahoo.com werden in Links umgewandelt.

    Ihre unterbrochenen Linien werden erhalten und neue Absätze beginnen mit Leerzeilen.

    Smileys wie :) werden automatisch in ihr grafisches Äquivalent umgewandelt.

    Sie können sogar HTML-Code einbetten - er bleibt erhalten.

    Mehr Informationen über Moodle-Text-Format.

2. HTML-Format

    Dieses Format setzt den Text in reinem HTML voraus. Wenn Sie den HTML-Editor verwenden, um Text zu bearbeiten, ist dies das Default-Format - alle Befehle in der Toolbar erstellen HTML für Sie.

    Selbst wenn Sie nicht mit dem Rich-Text-HTML-Editor arbeiten, können Sie HTML-Code in Ihrem Text benutzen und es kommt genau das heraus, was Sie beabsichtigen.

    Anders als beim Moodle-Auto-Format wird dann keine automatische Formatierung durchgeführt.

    Mehr Informationen über den Rich-Text-HTML-Editor

3. Reines Textformat

    Dieses Format empfiehlt sich, um große Mengen an Code oder HTML einzugeben, die genau so angezeigt werden sollen, wie Sie sie eingegeben haben.

    Leerzeichen und neue Zeilen werden umgesetzt. Sonst bleibt Ihr Text aber unberührt.

4. Markdown-Text-Format

    Das Markdown-Format erlaubt es recht einfach, sauber formatierte XHTML-Seiten zu erfassen. Sie können Text einfach in der gleichen Art und Weise schreiben wie z.B. E-Mails.

    Es eignet sich gut, um Text mit wenigen Formatierungen (z.B. Überschriften und Listen), aber ohne Bilder und Links zu erstellen.
