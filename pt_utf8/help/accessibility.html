Acessibilidade

Declaração da Acessibilidade

Este sítio foi testado em vários navegadores (browsers) e com software de leitura de ecrã.

O Moodle usa marcas semânticas para ajudar os utilizadores de leitores de ecrã e outros — os cabeçalhos do sítio, páginas e blocos usam <h1>, <h2>; os blocos da navegação são listas <ul>... e assim por diante.

O Moodle Segue as [Normas de Acessibilidade de Conteúdos, versão 1.0], de nível 1, da World Wide Web Consortium, a maioria das do nível 2 [ sobre a conformidade Dupla-A] e algumas de nível 3. Através destas directrizes esperamos cumprir com as leis locais a respeito do acesso para pessoas com deficiências. Actualmente estamos a trabalhar para melhorar a acessibilidade e a usabilidade do Moodle. Presentemente existem ainda algumas tabelas usadas para formatação, mas, em geral, a forma de apresentação é separada do conteúdo usando folhas de estilo.

Agradecemos quaisquer comentários, em particular relatórios de algum conteúdo inacessível. Por favor [ envie uma mensagem para o administrador].

Chaves de Acesso

Actualmente não se usam chaves de acesso, mas pretendemos usá-las numa versão futura do Moodle.

Apontadores

Há um apontador antes de cada bloco lateral, que permite que os utilizadores de leitores de ecrã avancem para o bloco seguinte. Note que o apontador é oculto nos navegadores gráficos.

Muitos apontadores têm atributos de título que os descrevem em maior detalhe, a menos que o texto do apontador descreva já por inteiro o destino (tal como o título de um artigo). Se um apontador tiver uma chave de acesso esta será anunciada no título.

Sempre que possível, os apontadores são escritos para fazer sentido fora do contexto. Não há nenhum pseudo apontador Javascript.

Imagens

Todas as imagens usam o atributo "alt" para fornecer o texto alternativo onde este for apropriado; as imagens que são puramente decorativas contêm um atributo nulo do "alt". As imagens, tais como os mapas, que apresentam informação complexa têm um atributo de "longdesc" ligando-os a uma descrição do conteúdo da imagem.

Conformidade com os padrões

Este sítio foi construído usando, para o conteúdo, XHTML válido, e CSS para a apresentação.
