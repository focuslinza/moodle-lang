Pagluluklok ng Moodle

Huwag masisiraan ng loob!

Ipinapaliwanag ng gabay na ito kung paano magluklok ng Moodle sa unang pagkakataon. Ang ilan sa mga hakbang ay maraming detalye. Ginawa ito upang masaklaw ang mayorya ng mga posibleng kaayusan ng web server, kaya't mukhang medyo mahaba at komplikado ang dokumentong ito. Huwag kang masisiraan ng loob. Kapag alam mo na kung paano ito gawin, mailuluklok mo na ang Moodle sa loob lamang ng ilang minuto!

Kung may mga nararanasan kang problema, pakibasa nang mabuti ang dokumentong ito - ang mga karaniwang problema ay masasagot dito. Kung may problema ka pa rin, makakahingi ka ng tulong sa Moodle Help

Ang isa pang opsiyon ay kumontak ka ng kumpanyang nagho-host ng web site na ganap na makapagmementina ng Moodle para sa iyo, para mabalewala mo na ang babasahing ito at dumiretso na sa pagtuturo!

 

Mga seksiyon ng dokumentong ito:

  1. Mga Kinakailangan
  2. Ilusong at kopyahin ang mga sako sa wastong lugar
  3. Balangkas ng site
  4. Patakbuhin ang iskrip na pangluklok para makalikha ng config.php
    • Suriin ang kaayusan ng web server
    • Paglikha ng datosan
    • Paglikha ng bugsok ng datos
  5. Tumungo sa pahinang pang-admin para maipagpatuloy ang pagsasaayos
  6. Isaayos ang cron
  7. Lumikha ng bagong kurso

 

1. Mga Kinakailangan

Ang Moodle ay pangunahing dinebelop sa Linux na gumagamit ng Apache, MySQL at PHP (na tinatawag ding LAMP na plataporma), bagama't tinetesting din nang regular sa PostgreSQL at sa Windows XP, Mac OS X at Netware 6 na mga sistemang pampatakbo

Ang mga kinakailangan ng Moodle ay ang mga sumusunod:

  1. Web server software. Ang ginagamit ng karamihang tao ay Apache, pero gagana rin nang maayos ang Moodle sa ilalim ng anumang web server na sumusuporta sa PHP, tulad ng IIS sa Windows na plataporma.
  2. PHP scripting language (version 4.1.0 o mas bago). Ang PHP 5 ay sinusuportahan na sa Moodle 1.4.
  3. gumaganang server ng datosan: ang MySQL o PostgreSQL ay ganap na sinusuportahan at inirerekomenda na gamitin sa Moodle.

Sinusuportahan ang mga ito ng karamihang web host bilang umiiral na kaayusan. Kung naka-sign-up ka sa isa sa iilang webhosts na hindi sinusuportahan ang mga katangiang ito, tanongin mo sila kung bakit, at pag-isipan mong maghanap ng ibang web host.

Kung gusto mong patakbuhin ang Moodle sa sarili mong kompyuter at parang masyadong mahirap sa iyo ang mga bagay na ito, pakitingnan ang gabay naming: Pagluluklok ng Apache, MySQL at PHP. Naglalahad ito ng mga hakbang na susundin upang mailuklok ang lahat ng ito sa karamihan ng mga popular na plataporma.

 

2. Ilusong at kopyahin ang mga sako sa wastong lugar

May dalawang paraan sa pagkuha ng Moodle, bilang isang siniksik na pakete at sa pamamagitan ng CVS. Ipinaliliwanag ito ng detalyado sa pahina na lusongan na:http://moodle.org/download/

Matapos ilusong at buklatin ang arkibo, o i-check out ang mga sako sa pamamagitan ng CVS, magkakaroon ka ng isang bugsok na may pangalang "moodle", na naglalaman ng ilang sako at polder.

Maaari mong ilagay ang buong polder sa bugsok ng "web server document" mo; kaya't ang magiging lokasyon ng site ay http://webservermo.com/moodle, o puwede mong kopyahin ang lahat ng nilalaman sa punong bugsok ng web server, kaya ang site ay magiging http://webservermo.com.

Kung inilulusong mo muna ang Moodle sa lokal mong kompyuter bago mo ito iahon sa web site mo, mas mabuting iahon ang buong arkibo bilang isang sako lamang, pagkatapos ay buklatin mo na lamang ito sa server. Kahit ang mga web hosting interpeys tulad ng Cpanel ay pinapahintulutan kang magbuklat ng mga arkibo sa "File Manager".

 

3. Balangkas ng site

Puwede mo nang laktawan ang seksiyong ito, gayunpaman narito ang mabilisang lagom ng mga nilalaman ng Moodle polder, para maging pamilyar ka dito:

config.php - naglalaman ng mga pangunahing kaayusan. Hindi kasama sa Moodle ang sakong ito - kailangan mo itong likhain.
install.php - ang iskrip na patatakbuhin mo upang malikha ang config.php
version.php - binibigyang depinisyon ang kasalukuyang bersiyon ng koda ng Moodle
index.php - ang unang pahina ng site

  • admin/ - koda para pamahalaan ang buong server
  • auth/ - mga plug-in na modyul para masino ang mga user
  • blocks/ - mga plug-in na modyul para sa maliliit na bloke sa gilid ng iba't-ibang pahina
  • calendar/ - lahat ng koda para sa pamamahala at pagdispley ng kalendaryo
  • course/ - koda para sa pagdispley at pamamahala ng mga kurso
  • doc/ - tulong na dokumentasyon para sa Moodle (eg ang pahinang ito)
  • files/ - koda para sa pagdispley at pamamahala ng mga inahon na file
  • lang/ - mga tekstong may sari-saring wika, isang bugsok bawat wika
  • lib/ - mga aklatan ng pinakasentrong koda ng Moodle code
  • login/ - mga koda para sa pagsasagawa ng paglagda at paglikha ng akawnt
  • mod/ - ang lahat ng pangunahing pangkursong modyul ng Moodle ay naririto
  • pix/ - mga karaniwang larawan ng site
  • theme/ - pakete/balat ng tema para mabago ang itsura ng site.
  • user/ - koda para magdispley at mapamahalaan ang mga tagagamit

 

4. Patakbuhin ang iskrip na pangluklok upang malikha ang config.php

Para mapatakbo ang iskip na pangluklok (install.php), pasukin lamang ang iyong punong URL ng Moodle sa pamamagitan ng web browser, o pasukin ang http://servermo/install.php nang direkta.

(Tatangkain ng Pangluklok na mag-set ng session cookie. Kung makakuha ka ng isang pop-up na babala sa browser mo, tanggapin mo ang cookie!)

Makikita ng Moodle na kailangang isaayos ito, at dadalhin ka sa ilang iskrin na tutulong sa iyong lumikha ng bagong sako ng kaayusan na may pangalang config.php. Sa katapusan ng proseso, tatangkain ng Moodle na isulat ang sako sa tamang lokasyon, kundi ay puwede mong pindutin ang isang buton para mailusong ito mula sa pangluklok, at pagkatapos ay iahon ang config.php sa punong bugsok ng Moodle sa server.

Habang ginagawa ang mga ito, susubukin ng panluklok ang kapaligiran ng server mo at magmumungkahi ng paraan kung paano malulutas ang anumang suliranin. Kung karaniwan lang ang problema, malulutas na ito sa pamamagitan ng mga mungkahing iyon, pero kung hindi malutas, tingnan mo sa ibaba ang iba pang impormasyon hinggil sa mga kadalasang pinagmumulan ng iyong mga problema.

4.1 Pangkalahatang kaayusan ng web server

Una, tiyakin na ginagamit ng web server mo ang index.php bilang umiiral na pahina (marahil ay kasama ng index.html, default.htm at iba pa).

Sa Apache, magagawa ito sa pamamagitan ng DirectoryIndex na parameter sa iyong sakong httpd.conf. Ganito ang itsura ng httpd.conf ko: 

DirectoryIndex index.php index.html index.htm

Basta tiyaking nasa listahan ang index.php (at mas maigi kung nasa unahan ng listahan, para maging masinop).

Pangalawa, kung ang ginagamit mo ay Apache 2, buhayin mo ang baryabol na AcceptPathInfo , na siyang nagpapahintulot sa mga iskrip na mapasahan ng mga argumento tulad ng http://server/file.php/arg1/arg2. Mahalaga ito para mapahintulutan ang mga relatibong link ng mga rekurso mo, at nagpapahusay sa paggamit ng mga tao sa Moodle web site mo. Maaari mo itong buhayin sa pamamagitan ng pagdaragdag ng mga linyang ito sa sakong httpd.conf mo. 

AcceptPathInfo on

Pangatlo, kailangan ng Moodle na maging aktibo ang ilang PHP setting para gumana ito. Sa karamihang server ito na ang mga umiiral na setting. Gayunpaman, may ilang PHP server (at ilang mas bagong bersiyon ng PHP) na iba ang setting. Itinatakda ito sa sakong pangkaayusan ng PHP (na karaniwang tinatawag na php.ini):

magic_quotes_gpc = 1    (mas mabuti pero hindi kinakailangan)
magic_quotes_runtime = 0    (kailangan)
file_uploads = 1
session.auto_start = 0
session.bug_compat_warn = 0

Kung hindi mo puwedeng galawin ang httpd.conf o php.ini sa server mo, o kung ang Moodle mo ay nasa server na may ibang aplikasyon na nangangailangan ng ibang setting, walang problema, maaari mong PANAIGAN ang mga umiiral na setting.

Para magawa ito, kailangan mong lumikha ng sako na ang pangalan ay .htaccess sa punong bugsok ng Moodle na naglalaman ng mga linya na katulad ng sumusunod. Gagana lamang ito sa mga Apache server at tangi kung ang Overrides ay pinapahintulutan sa pangunahing kaayusan. 

DirectoryIndex index.php index.html index.htm

<IfDefine APACHE2>
     AcceptPathInfo on
</IfDefine>

php_flag magic_quotes_gpc 1
php_flag magic_quotes_runtime 0
php_flag file_uploads 1
php_flag session.auto_start 0
php_flag session.bug_compat_warn 0

Maaari mo ring itakda ang maksimum na laki ng maiaahon na sako: 

LimitRequestBody 0
php_value upload_max_filesize 2M
php_value post_max_size 2M

Ang pinakamadaling gawin ay kopyahin mo na lamang ang sampol na sako sa lib/htaccess at iedit ito ayon sa iyong pangangailangan. May laman itong dagdag na panuto. Halimbawa, sa isang Unix shell: 

cp lib/htaccess .htaccess

 

4.2 Paglikha ng datosan

Kailangan mong lumikha ng datosan na walang laman (eg "moodle") sa sistemang datosan mo na may espesyal na tagagamit (eg "moodleuser"). Ang tagagamit na ito ay dapat na may karapatang pumasok sa datosan (at tanging datosan na yaon). Maaari mong gamitin ang "root" na tagagamit kung gusto mo ng pansubok lamang na server, pero hindi ito inirerekomenda para sa isang sistemang pamproduksiyon: Kapag natuklasan ng mga hacker ang kontrasenyas nito, hindi lamang ang isang datosan ang manganganib kundi ang buong sistemang datosan.

Kung gumagamit ka naman ng webhost, malamang ay may web interpeys na control panel ka para makalikha ng datosan.

Ang pinakapopular na control panel ay ang sistemang Cpanel. Ganito ang paglikha ng datosan sa Cpanel,

  1. Iklik ang "MySQL Databases" na ikon.
  2. Iteklado ang "moodle" sa pitak ng datosan at iklik ang "Add Database".
  3. Iteklado ang bansag at kontrasenyas (huwag iyong ginagamit sa ibang bagay) sa naaayong pitak at iklik ang "Add User".
  4. Pagkatapos ay gamitin ang "Add User to Database" na buton para mabigyan ang bagong akawnt ng tagagamit na ito ng "ALL" na karapatan sa bagong datosan.
  5. Pansinin na ang bansag at pangalan ng datosan ay maaaring inunlapian ng pangalan ng akawnt mo sa Cpanel. Kapag ipinasok mo ang impormasyong ito sa pangluklok ng Moodle - gamitin mo ang buong pangalan.

Kung magagamit mo ang command line ng Unix, puwede mong gawin ang mga binanggit sa itaas sa pamamagitan ng pagteklado ng mga atas.

Heto ang ilang halimbawa ng mga atas sa Unix command line para sa MySQL: 

   # mysql -u root -p
   > CREATE DATABASE moodle; 
   > GRANT SELECT,INSERT,UPDATE,DELETE,CREATE,DROP,INDEX,ALTER ON moodle.* 
           TO moodleuser@localhost IDENTIFIED BY 'yourpassword'; 
   > quit 
   # mysqladmin -p reload

At ilang halimbawang atas sa command line para sa PostgreSQL:

   # su - postgres
   > psql -c "create user moodleuser createdb;" template1
   > psql -c "create database moodle;" -U moodleuser template1
   > psql -c "alter user moodleuser nocreatedb;" template1

 

4.3 Paglikha ng bugsok ng datos

Mangangailangan din ang Moodle ng espasyo sa hard disk ng server mo para mapag-imbakan ng mga inahon na sako, tulad ng mga dokumento para sa kurso at mga larawan ng tagagamit.

Pipilitin ng panluklok ng Moodle na likhain ang bugsok na ito, pero kapag nabigo ito kailangan mong likhain ang bugsok nang mano-mano.

Para maingatan ang seguridad, mas mabuti na HUWAG pahintulutang mapasok nang direkta mula sa web ang bugsok na ito. Ang pinakamadaling paraan para magawa ito ay ilagay ito sa LABAS ng bugsok na pangweb , pero kung kailangan mong ilagay ito sa bugsok na pangweb ay protektahan mo ito sa pamamagitan ng paglikha ng sako sa bugsok ng datos na tinatawag na .htaccess, na naglalaman ng sumusunod na linya: 

deny from all
AllowOverride None

Para matiyak na kaya ng Moodle na magsilid ng inahong sako sa bugsok na ito, tingnan mo kung ang web server software (eg Apache) ay may permisong bumasa, sumulat at magpatakbo ng program sa bugsok na ito.

Sa mga makinang Unix, ay pinapangalanan ang may-ari ng bugsok ng tulad ng "nobody" o "apache", pagkatapos ay binibigyan ang tagagamit na ito ng kinakailangang permiso sa pagbasa, pagsulat at pagpapatakbo ng program sa bugsok.

Sa Cpanel system, maaari mong gamitin ang "File Manager" para mahanap ang polder, iklik ito, pagkatapos ay piliin ang "Change Permissions". Sa maraming pinagsasaluhang hosting server, marahil ay kailangan mong limitahan ang lahat ng file access sa group mo (upang maiwasan ang pagtingin ng ilang kostumer ng webhost o pagbabago nila, ng mga sako mo), pero bigyan mo ng ganap na karapatang bumasa/sumulat ang iba pa (na magpapahintulot sa web server na magamit ang mga sako mo).

Makipag-usap ka sa administrador ng server mo, kung namomoroblema ka sa pagsasaayos nito nang may sapat na seguridad. Ang ilang site na gumagamit ng katangiang pang-PHP na tinatawag na "Safe Mode" ay maaaring magtakda ng kondisyon na tanging ang administrador ang makalilikha nang wasto ng bugsok na ito para sa iyo.

5. Tumungo sa pahinang pang-admin para maituloy ang pagsasaayos

Kapag ang payak na config.php ay nalikha nang wasto sa naunang hakbang, mapupunta ka sa pahina ng "admin" sa unang beses na tangkain mong pumasok sa harapang pahina ng site para matapos ang pagsasaayos.

Sa unang beses na pasukin mo ang pahina ng admin na ito, papakitaan ka ng isang GPL "shrinkwrap" na kasunduan na kailangan mong sang-ayunan bago ka makapagpatuloy sa pagsasaayos.

Ngayon, magsisimula nang iayos ng Moodle ang datosan mo at lilikha ito ng mga teybol para maimbakan ng datos. Una, lilikhain ang mga punong teybol ng datosan. Makakakita ka ng ilang SQL na pahayag, na susundan ng mga mensahe ng estado (na kulay berde o pula) na may ganitong itsura:

CREATE TABLE course ( id int(10) unsigned NOT NULL auto_increment, category int(10) unsigned NOT NULL default '0', password varchar(50) NOT NULL default '', fullname varchar(254) NOT NULL default '', shortname varchar(15) NOT NULL default '', summary text NOT NULL, format tinyint(4) NOT NULL default '1', teacher varchar(100) NOT NULL default 'Teacher', startdate int(10) unsigned NOT NULL default '0', enddate int(10) unsigned NOT NULL default '0', timemodified int(10) unsigned NOT NULL default '0', PRIMARY KEY (id)) TYPE=MyISAM

TAGUMPAY

...at gayon nang gayon, tapos ay susundan ng: Matagumpay na naisaayos ang mga pangunahing datosan.

Kapag hindi mo nakita ang mga ito, maaaring nagkaproblema sa datosan o sa kaayusan na itinakda sa config.php. Tingnan kung ang PHP ay hindi nasa restricted na "Safe Mode" (minsan binubuhay ng mga komersiyal na web host ang safe mode). Maaari mong tsekin ang mga baryabol na PHP sa pamamagitan ng paglikha ng maliit na sako na naglalaman ng <?php phpinfo() ?> at pagtingin dito sa pamamagitan ng browser. Suriin lahat ito at tangkain muli ang paggamit ng pahinang ito.

Mag-iskrol pababa sa dulo ng pahina at pindutin ang "Ituloy" na link.

Makakakita ka ng porma kung saan maitatakda mo ang marami pang baryabol para sa pagsasaayos ng luklok mo, tulad ng pinaiiral na wika, SMTP host at iba pa. Huwag kang mag-alala kung hindi mo magawa ang lahat ng gusto mo sa ngayon - maaari ka namang bumalik dito at iedit ang iba pang hindi mo naiayos sa pamamagitan ng pang-admin na intepeys. Ang mga pinaiiral ay dinisenyo na maging kapakipakinabang at ligtas para sa karamihang site. Mag-iskrol pababa at iklik ang "Isilid ang pagbabago".

Kung (at tangi kung) hindi ka makaalis sa pahinang ito, at hindi ka makapagpatuloy, may problema ang server mo na ang tawag ko ay problemang "buggy referrer". Madali itong kumpunihin: patayin mo lamang ang "secureforms" na setting, pagkatapos ay tangkain mong magpatuloy muli.

Pagkatapos ay makakakita ka ng marami pang pahina na naglalathala ng maraming mensahe ng estado habang isinasaayos nila ang lahat ng teybol na kinakailangan ng iba't-ibang modyul ng Moodle. Tulad din ng nauna, dapat ay kulay berde ang mga ito.

Mag-iskrol sa pinaka-ibaba ng pahina at pindutin ang "Ituloy" na link.

Ang susunod na pahina ay isang porma na pagtatakdaan mo ng mga parameter para sa Moodle site at sa pambungad na pahina, tulad ng pangalan, anyo, deskripsiyon at iba pa. Punan ang mga kahon nito (puwede mo itong balikan anumang oras para baguhin), tapos ay pindutin ang "Isilid ang mga pagbabago".

Bilang panghuli, hihilingin kang lumikha ng isang pinakamataas-na-antas na tagagamit na administrador para mapasok ang mga pahina ng admin. Punan ang mga kinakailangang detalye ng pangalan mo, email atbp., tapos ay iklik ang "Isilid ang mga pagbabago". Hindi lahat ng pitak ay kailangan, pero kapag may mahalagang pitak kang nakaligtaan, huhudyatan kang muli para sagutan ang mga ito.

Tiyakin na natatandaan mo ang bansag at kontrasenyas na pinili mo para sa akawnt ng tagagamit na administrador, dahil kailangan ito para mapasok ang pahinang pang-administrasyon sa mga susunod na pagkakataon.

(Kung dahil sa anumang bagay ay nahinto ang pagluluklok mo, o kung nagka-error na pansistema, na pumigil sa iyong lumagda sa pamamagitan ng akawnt na admin, kadalasan ay makakalagda ka sa pamamagitan ng umiiral na bansag na "admin", na may kontrasenyas na "admin".)

Kapag nagtagumpay ka na, ibabalik ka sa tahanang pahina ng bago mong site! Pansinin mo ang mga link na pang-administrador sa may ibabang-kaliwa ng pahina (lumilitaw din ang mga aytem na ito sa hiwalay na pahinang pang-Admin) - nakikita mo lamang ang mga aytem na ito dahil nakalagda ka bilang tagagamit na admin. Lahat nang iba pang pamamahala ng Moodle na kailangan mong gawin ay maaari nang gawin sa pamamagitan ng menung ito, tulad ng:

  • paglikha at pagbura ng mga kurso
  • paglikha at pag-edit ng mga akawnt ng tagagamit
  • pamamahala ng mga akawnt ng guro
  • pagbabago ng mga setting na laganap sa buong site tulad ng tema, atbp.

Pero hindi pa tapos ang pagluluklok mo! May isa pang napakahalagang bagay na kailangang gawin (tingnan ang sususunod na seksiyon hinggil sa cron).

 

6. Isaayos ang cron -- IMPORTANTE!

Kailangan ng pagmamanman sa ilang modyul ng Moodle upang maisakatuparan ang mga gawain nito. Halimbawa, kailangan tingnan ng Moodle ang mga talakayan upang mai-email nito ang mga kopya ng ipinaskil sa mga kasali.

Ang iskrip na gumagawa ng lahat ng ito ay nasa bugsok na admin, at ang pangalan ay cron.php. Kaya lamang, hindi nito kayang patakbuhin ang sarili nito, kaya kailangan mong magsaayos ng mekanismo na magpapatakbo ng iskrip na ito nang regular (eg tuwing ikalima o ikasampung minuto). Mistula ngayon itong "pintig-ng-puso" upang maisakatuparan ng iskrip ang mga gawain nito sa panahong itinatakda ng bawat modyul. Ang ganitong uri ng regular na mekanismo ay tinatawag na serbisyong cron .

Gusto kong bigyang pansin na hindi kailangang ang makinang nagpapatakbo ng Moodle ang makinang gagawa ng cron. Halimbawa, kung limitado ang serbisyong web hosting mo, at wala itong serbisyong cron, puwede mong patakbuhin ang cron sa isa pang server o sa kompyuter mo sa bahay. Ang mahalaga lamang ay mapatakbo ang cron.php nang regular.

Magaang lamang ang paggamit ng kompyuter ng iskrip na ito, kaya ang 5 minuto ay makatwiran na, pero kung nag-aalala ka sa kinakain nitong rekurso ng server, maaari mong pahabain ang agwat ng pagtakbo sa mga 15 minuto o kahit 30 minuto. Mas mabuting huwag masyadong pahabain ang agwat na ito dahil ang pagbalam sa pagpapadala-ng-liham ay makapagpapabagal sa aktibidad ng kurso.

Una, subukin kung tumatakbo ang iskrip sa pamamagitan ng pagpapatakbo nito nang direkta sa pamamagitan ng iyong browser: 

http://halimbawa.com/moodle/admin/cron.php

Pagkatapos, kailangan mong magsaayos ng isang paraan ng pagpapatakbo ng script nang awtomatiko at regular.

Sa sistemang Windows

Ang pinasimpleng paraan ay gamitin ang maliit na paketeng ito moodle-cron-for-windows.zip na magpapadali sa gawaing ito sa pamamagitan ng pagluklok ng isang maliit na serbisyong pang-Windows. Patakbuhin mo ito at kalimutan na!

Sa serbisyong web hosting

Maaaring may pahinang pangweb ang control panel mo para maisaayos ang prosesong cron na ito. Halimbawa, sa sistemang Cpanel, maghanap ng buton na may pangalang "Cron jobs". Doon mailalagay mo ang mga Unix command na nakalista sa ibaba.

Paggamit ng command line sa Unix

Maraming command line program na magagamit mo sa pagtawag sa pahinang may iskrip mula sa command line. Maaring hindi lahat nang ito ay nasa isang server.

Halimbawa, maaari mong gamitin ang isang Unix utility tulad ng 'wget': 

wget -q -O /dev/null http://halimbawa.com/moodle/admin/cron.php

Pansinin na sa halimbawang ito ang output ay itinatapon (sa /dev/null).

Ganito rin ang paraan kung gagamitin mo ang lynx: 

lynx -dump http://halimbawa.com/moodle/admin/cron.php > /dev/null

O kaya'y maaari kang gumamit ng standalone na bersiyon ng PHP, na kinompayl para tumakbo sa command line. Ang kaigihan ng ganitong paraan ay hindi mapupuno ang web server log mo ng paulit-ulit na paghiling sa cron.php. Ang kahinaan ng paraang ito ay kailangan mo ng command-line na bersiyon ng php. 

/opt/bin/php /web/moodle/admin/cron.php

Paggamit ng crontab program sa Unix

Ang ginagawa lang naman ng Cpanel ay maglagay ng interpeys na pangweb para sa isang kagamitang pang-Unix na tinatawag na crontab. Kung may command line ka, maaari mong isaayos ang crontab sa sarili mo sa pamamagitan ng atas na ito: 

crontab -e

pagkatapos ay idagdag ang isa sa mga atas sa itaas tulad ng:

*/5 * * * * wget -q -O /dev/null http://halimbawa.com/moodle/admin/cron.php

Kadalasan, ipapasa ka ng "crontab" command sa 'vi' editor. Makakapasok ka sa "insert mode" sa pamamagitan ng pagpindot ng "i", tapos ay iteklado mo ang linya sa itaas, tapos ay lumabas sa modong insert sa pamamagitan ng pagpindot ng ESC. Makapagsisilid at makakalabas ka sa pamamagitan ng pagteklado ng ":wq", o umayaw nang hindi nagsisilid ng anuman sa pamamagitan ng ":q!" (alisin ang mga panipi).

 

7. Lumikha ng bagong kurso

Ngayo't tumatakbo na nang maayos ang Moodle, bakit di ka lumikha ng bagong kurso na mapaglalaruan.

Piliin ang "Lumikha ng bagong kurso" sa pahinang Admin (o sa pang-admin na link sa tahanang pahina).

Punan ang porma, pag-ingatan mo ang pagtukoy sa anyo ng kurso. Hindi mo kailangang busisiin ang detalye sa panahong ito, dahil lahat ay maaaring baguhin sa ibang pagkakataon ng mga guro. Tandaan na ang dilaw na ikon ng tulong ay nakakalat para magbigay ng nasa-kontekstong tulong sa anumang aspeto.

Pindutin ang "Isilid ang mga pagbabago", at ipapasa ka sa isang bagong porma kung saan puwede kang magtakda ng guro sa kurso. Maaari mo lamang idagdag sa pormang ito ang mga akawnt ng tagagamit na nalikha na - kung gusto mong lumikha ng bagong akawnt ng guro, sabihin mo sa guro na lumikha ng akawnt para sa sarili niya (tingnan ang pahinang panglagda), o lumikha ng isa para sa kanila sa pamamagitan ng "Magdagdag ng bagong tagagamit" sa pahinang pang-Admin.

Pag natapos na ito, puwede nang lagyan ng pasadyang katangian ang kurso, at magagawa ito sa pamamagitan ng "Mga Kurso" na link sa tahanang pahina.

Tingnan ang "Manwal ng Guro" para sa mga detalye ng paglikha ng kurso.

 

Masayang pag-aaral at Maligayang pagmu-Moodle!

Kung nagustuhan mo ang Moodle, pag-isipan din po ninyo ang pagdodoneyt para matulungan kami sa gastusin!
