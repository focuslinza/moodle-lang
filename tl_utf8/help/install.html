Pagluluklok ng Moodle

Huwag masisiraan ng loob!

Ipinapaliwanag ng gabay na ito kung paano magluklok ng Moodle sa unang pagkakataon. Ang ilan sa mga hakbang ay maraming detalye upang masakop ang mayorya ng mga posibleng kaayusan ng web server, kaya't mukhang medyo mahaba at komplikado ang dokumentong ito. Huwag masisiraan ng loob, kapag alam na ninyo kung paano ito gawin, mailuluklok ninyo ang Moodle sa loob ng ilang minuto!

Kung may mga nararanasan kang problema, pakibasa nang mabuti ang dokumentong ito - ang mga karaniwang problema ay nasasagot dito. Kung may problema ka pa rin, makakahingi ka ng tulong sa Moodle Help

Ang isa pang opsiyon ay kumontak ka ng kumpanyang nagho-host ng web site na ganap na makapagmementina ng Moodle para sa iyo, para mabalewala mo na ang babasahing ito at dumiretso na sa pagtuturo!

 

Mga seksiyon ng dokumentong ito:

  1. Mga Kinakailangan
  2. Ilusong at kopyahin ang mga file sa wastong lugar
  3. Balangkas ng site
  4. Patakbuhin ang iskrip na pangluklok para makalikha ng config.php
    • Suriin ang kaayusan ng web server
    • Paglikha ng datosan
    • Paglikha ng direktoryo ng datos
  5. Tumungo sa pahinang pang-admin para maipagpatuloy ang pagsasaayos
  6. Isaayos ang cron
  7. Lumikha ng bagong kurso

 

1. Mga Kinakailangan

Ang Moodle ay pangunahing dinebelop sa Linux na gumagamit ng Apache, MySQL at PHP (na tinatawag ding LAMP na plataporma), bagama't tinetesting din nang regular sa PostgreSQL at sa Windows XP, Mac OS X at Netware 6 na mga sistemang pampatakbo

Ang mga kinakailangan ng Moodle ay ang mga sumusunod:

  1. Web server software. Ang ginagamit ng karamihang tao ay Apache, pero gagana rin nang maayos ang Moodle sa ilalim ng anumang web server na sumusuporta sa PHP, tulad ng IIS sa Windows na plataporma.
  2. PHP scripting language (version 4.1.0 o mas bago). Ang PHP 5 ay sinusuportahan na sa Moodle 1.4.
  3. gumaganang datosang server: MySQL o PostgreSQL ay ganap na sinusuportahan at inirerekomenda na gamitin sa Moodle.

Sinusuportahan ng karamihang web host ang lahat ng ito nang umiiral. Kung naka-sign-up ka sa isa sa iilang webhosts na hindi sinusuportahan ang mga katangiang ito, tanongin mo sila kung bakit, at pag-isipan mong maghanap ng ibang web host.

Kung gusto mong patakbuhin ang Moodle sa sarili mong kompyuter at parang masyadong mahirap ang mga bagay na ito, pakitingnan ang gabay naming: Pagluluklok ng Apache, MySQL at PHP. Naglalahad ito ng mga hakbang na susundin upang mailuklok ang lahat ng ito sa karamihan sa mga popular na plataporma.

 

2. Ilusong at kopyahin ang mga file sa wastong lugar

May dalawang paraan sa pagkuha ng Moodle, bilang isang siniksik na pakete at sa pamamagitan ng CVS. Ipinaliliwanag ito ng detalyado sa pahina na pinaglulusongan na:http://moodle.org/download/

Matapos ilusong at buklatin ang arkibo, o makuha ang mga file sa pamamagitan ng CVS, magkakaroon ka ng isang direktoryong may pangalang "moodle", na naglalaman ng ilang file at folder.

Maaari mong ilagay ang buong folder sa direktoryo ng "web server document" mo, kaya ang magiging lokasyon ng site ay http://webservermo.com/moodle, o puwede mong kopyahin ang lahat ng nilalaman sa punong direktoryo ng web server, kaya ang site ay magiging http://webservermo.com.

Kung inilulusong mo ang Moodle sa lokal mong kompyuter bago mo ito iahon sa web site mo, mas mabuting iahon ang buong arkibo bilang isang file lamang, pagkatapos ay buklatin ito sa server. Kahit ang mga web hosting interface tulad ng Cpanel ay pinapahintulutan kang mag-uncompress ng mga arkibo sa "File Manager".

 

3. Balangkas ng site

Puwede mo nang laktawan ang seksiyong ito, gayunpaman narito ang mabilisang lagom ng mga nilalaman ng Moodle folder, para maging pamilyar ka sa program:

config.php - naglalaman ng mga pangunahing kaayusan. Hindi kasama sa Moodle ang file na ito - kailangan mo itong likhain.
install.php - ang iskrip na patatakbuhin mo upang malikha ang config.php
version.php - binibigyang depinisyon ang kasalukuyang bersiyon ng Moodle code
index.php - ang pambungad na pahina ng site

  • admin/ - code para pamahalaan ang buong server
  • auth/ - mga plugin module para i-authenticate ang mga user
  • blocks/ - mga plugin module para sa maliliit na bloke sa gilid sa iba't-ibang pahina
  • calendar/ - lahat ng code para sa pamamahala at pagdispley ng kalendaryo
  • course/ - code para sa pagdispley at pamamahala ng mga kurso
  • doc/ - tulong na dokumentasyon para sa Moodle (eg ang pahinang ito)
  • files/ - code para sa pagdispley at pamamahala ng mga inahon na file
  • lang/ - mga tekstong may sari-saring wika, isang direktoryo bawat wika
  • lib/ - mga library ng core na Moodle code
  • login/ - code para sa pag-handle ng lagda at paglikha ng akawnt
  • mod/ - lahat ng pangunahing Moodle course module ay naririto
  • pix/ - generic na mga larawan ng site
  • theme/ - pakete/balat ng tema para mabago ang itsura ng site.
  • user/ - code para magdispley at pamahalaan ang mga tao

 

4. Patakbuhin ang iskrip na pangluklok upang malikha ang config.php

Para mapatakbo ang iskip na pangluklok (install.php), pasukin lamang ang inyong punong URL ng Moodle sa pamamagitan ng web browser, o pasukin ang http://servermo/install.php nang direkta.

(Tatangkain ng Pangluklok na mag-set ng session cookie. Kung makakuha ka ng isang pop-up na babala sa browser mo, tiyakin na tanggapin ang cookie!)

Makikita ng Moodle na kailangang isaayos ito, at dadalhin ka sa ilang iskrin na tutulong sa iyong lumikha ng bagong configuration file na may pangalang config.php. Sa katapusan ng proseso, tatangkain ng Moodle na isulat ang file sa tamang lokasyon, kundi man puwede mong pindutin ang isang buton para mailusong ito mula sa pangluklok, at pagkatapos ay iahon ang config.php sa punong Moodle direktoryo sa server.

Habang ginagawa ang mga ito, susubukin ng installer ang server environment mo at magmumungkahi ng paraan kung paano malulutas ang anumang suliranin. Para sa karaniwang problema, sapat na ang mga mungkahing ito para malutas ang mga iyon, pero kung hindi malutas, tingnan mo sa ibaba ang iba pang impormasyon hinggil sa mga kadalasang nagbibigay sa iyo ng problema.

4.1 Pangkalahatang kaayusan ng web server

Una, tiyakin na nakaayos ang web server mo na gamitin ang index.php bilang umiiral na pahina (marahil ay kasama ng index.html, default.htm at iba pa).

Sa Apache, magagawa ito sa pamamagitan ng DirectoryIndex na parameter sa inyong httpd.conf file. Ganito ang karaniwang itsura ng sa akin: 

DirectoryIndex index.php index.html index.htm

Basta tiyaking nasa listahan ang index.php (at mas maigi kung nasa unahan ng listahan, para sa kasinupan).

Pangalawa, kung ginagamit mo ang Apache 2, buhayin mo ang AcceptPathInfo na baryabol, na nagpapahintulot sa mga iskrip na magpasa ng mga argument tulad ng http://server/file.php/arg1/arg2. Mahalaga ito para mapahintulutan ang mga relatibong link sa pagitan ng mga rekurso mo, at nagpapahusay sa paggamit ng mga tao sa Moodle web site mo. Maaari mo itong buhayin sa pamamagitan ng pagdaragdag ng mga linyang ito sa httpd.conf file mo. 

AcceptPathInfo on

Pangatlo, kailangan ng Moodle na maging aktibo ang ilang PHP setting para gumana ito. Sa karamihang server ito na ang mga umiiral na setting. Gayunpaman, may ilang PHP server (at ilang mas bagong bersiyon ng PHP) na iba ang setting. Idini-define ito sa configuration file ng PHP (na karaniwang tinatawag na php.ini):

magic_quotes_gpc = 1    (mas mabuti pero hindi kinakailangan)
magic_quotes_runtime = 0    (kailangan)
file_uploads = 1
session.auto_start = 0
session.bug_compat_warn = 0

Kung wala kang access sa httpd.conf o php.ini sa server mo, o ang Moodle mo ay nasa server na may ibang aplikasyon na nangangailangan ng ibang setting, hindi ito problema, maaari mong PANAIGAN ang mga umiiral na setting.

Para magawa ito, kailangan mong lumikha ng file na ang pangalan ay .htaccess sa punong direktoryo ng Moodle na naglalaman ng mga linya na katulad ng sumusunod. Gagana lamang ito sa mga Apache server at tangi kung ang Overrides ay pinapahintulutan sa pangunahing kompigurasyon. 

DirectoryIndex index.php index.html index.htm

<IfDefine APACHE2>
     AcceptPathInfo on
</IfDefine>

php_flag magic_quotes_gpc 1
php_flag magic_quotes_runtime 0
php_flag file_uploads 1
php_flag session.auto_start 0
php_flag session.bug_compat_warn 0

Maaari mo ring itakda ang maksimum na laki ng inaahon na file: 

LimitRequestBody 0
php_value upload_max_filesize 2M
php_value post_max_size 2M

Ang pinakamadaling gawin ay kopyahin mo na lamang ang sampol na file sa lib/htaccess at iedit ito ayon sa iyong pangangailangan. May laman itong dagdag na panuto. Halimbawa, sa isang Unix shell: 

cp lib/htaccess .htaccess

 

4.2 Paglikha ng datosan

Kailangan mong lumikha ng walang laman na datosan (eg "moodle") sa sistemang datosan mo na may espesyal na tao (eg "moodleuser") na may karapatang pumasok sa datosan (at tanging datosan na yaon). Maaari mong gamitin ang taong "root" kung gusto mo ng sinusubok na server, pero hindi ito inirerekomenda para sa isang sistemang pamproduksiyon: Kapag natuklasan ng mga hacker ang kontrasenyas, hindi lamang ang isang datosan ang nasa panganib kundi ang buong sistemang datosan.

Kung gumagamit ka ng webhost, malamang ay may web interface na control panel ka para sa paglikha ng datosan.

Ang pinakapopular sa mga ito ay ang sistemang Cpanel. Para makalikha ng datosan sa Cpanel,

  1. Iklik ang "MySQL Databases" na ikon.
  2. Iteklado ang "moodle" sa pitak ng datosan at iklik ang "Add Database".
  3. Iteklado ang bansag at kontrasenyas (huwag iyong ginagamit sa ibang bagay) sa naaayong pitak at iklik ang "Add User".
  4. Pagkatapos ay gamitin ang "Add User to Database" na buton para mabigyan ang bagong akawnt ng tao na ito ng "ALL" na karapatan sa bagong datosan.
  5. Pansinin na ang bansag at ngalan ng datosan ay maaaring inuunlapian ng pangalan ng akawnt mo sa Cpanel. Kapag ipinasok mo ang impormasyong ito sa pangluklok ng Moodle - gamitin ang buong pangalan.

Kung may access ka sa Unix command line, puwede mong gawin ang binanggit sa itaas sa pamamagitan ng pagteklado ng mga command.

Heto ang ilang halimbawa ng mga Unix command line para sa MySQL: 

   # mysql -u root -p
   > CREATE DATABASE moodle; 
   > GRANT SELECT,INSERT,UPDATE,DELETE,CREATE,DROP,INDEX,ALTER ON moodle.* 
           TO moodleuser@localhost IDENTIFIED BY 'yourpassword'; 
   > quit 
   # mysqladmin -p reload

At ilang halimbawang command line para sa PostgreSQL:

   # su - postgres
   > psql -c "create user moodleuser createdb;" template1
   > psql -c "create database moodle;" -U moodleuser template1
   > psql -c "alter user moodleuser nocreatedb;" template1

 

4.3 Paglikha ng direktoryo ng datos

Mangangailangan din ang Moodle ng espasyo sa hard disk ng server mo para mapag-imbakan ng mga inahon na file, tulad ng mga dokumento para sa kurso at mga larawan ng user.

Pipilitin ng Moodle installer na likhain ang direktoryong ito para sa iyo, pero kapag nabigo ito kailangan mong likhain ang direktoryo nang mano-mano.

Para maingatan ang seguridad, mas mabuti na HUWAG gawing mapapasok nang direkta mula sa web ang direktoryong ito. Ang pinakamadaling paraan para magawa ito ay ilagay ito sa LABAS ng direktoryong pangweb , pero kung kailangan mong ilagay ito sa direktoryong pangweb ay protektahan mo ito sa pamamagitan ng paglikha ng file sa direktoryo ng datos na tinatawag na .htaccess, na naglalaman ng sumusunod na linya: 

deny from all

Para matiyak na kaya ng Moodle na magsave ng inahon na file sa direktoryong ito, tingnan kung ang web server software (eg Apache) ay may permisong bumasa, sumulat at magpatakbo ng program sa direktoryong ito.

Sa mga makinang Unix, ang ibig sabihin nito ay ang pagseset ng may-ari ng direktoryo na tulad ng "nobody" o "apache", at pagbibigay ng kinakailangang permiso ng user na ito upang bumasa, sumulat at magpatakbo ng program.

Sa Cpanel system, maaari mong gamitin ang "File Manager" para mahanap ang folder, iklik ito, pagkatapos ay piliin ang "Change Permissions". Sa maraming pinagsasaluhang hosting server, marahil ay kailangan mong limitahan ang lahat ng file access sa "group" mo (upang maiwasan ang pagtingin o pagbabago ng mga file mo ng ibang kostumer ng webhost), pero bigyan mo ng ganap na karapatang bumasa/sumulat ang iba pa (na magpapahintulot sa web server na magamit ang file mo).

Makipag-usap ka sa administrador ng server mo, kung namomoroblema ka sa pagseset-up nito nang may sapat na seguridad. Ang ilang site na gumagamit ng katangiang pang-PHP na tinatawag na "Safe Mode" ay maaaring magtakda ng kondisyon na tanging ang administrador ang makalilikha nang wasto ng direktoryong ito para sa iyo.

5. Tumungo sa pahinang pang-admin para makapagpatuloy sa pagsasaayos

Kapag ang payak na config.php ay nalikha nang wasto sa naunang hakbang, mapupunta ka sa pahina ng "admin" sa unang beses na tangkain mong pasukin ang harapang pahina ng site para matapos ang pagsasaayos.

Sa unang beses na pasukin mo ang pahina ng admin na ito, papakitaan ka ng isang GPL "shrinkwrap" na kasunduan na kailangan mong sang-ayunan bago ka makapagpatuloy sa pagsasaayos.

Ngayon, magsisimula nang iayos ng Moodle ang datosan mo at lilikha ito ng mga teybol para maimbakan ng datos. Una, lilikhain ang mga punong teybol ng datosan. Makakakita ka ng ilang SQL na pahayag, na susundan ng mga mensahe ng estado (na kulay berde o pula) na ganito ang itsura:

CREATE TABLE course ( id int(10) unsigned NOT NULL auto_increment, category int(10) unsigned NOT NULL default '0', password varchar(50) NOT NULL default '', fullname varchar(254) NOT NULL default '', shortname varchar(15) NOT NULL default '', summary text NOT NULL, format tinyint(4) NOT NULL default '1', teacher varchar(100) NOT NULL default 'Teacher', startdate int(10) unsigned NOT NULL default '0', enddate int(10) unsigned NOT NULL default '0', timemodified int(10) unsigned NOT NULL default '0', PRIMARY KEY (id)) TYPE=MyISAM

TAGUMPAY

...at patuloy, tapos ay susundan ng: Matagumpay na naisaayos ang mga pangunahing datosan.

Kapag hindi mo nakita ang mga ito, maaaring nagkaproblema sa datosan o sa configuration setting na itinakda sa config.php. Tingnan kung ang PHP ay hindi nasa restricted na "Safe Mode" (minsan binubuhay ng mga komersiyal na web host ang safe mode). Maaari mong tsekin ang mga PHP variable sa pamamagitan ng paglikha ng maliit na file na naglalaman ng <?php phpinfo() ?> at pagtingin dito sa pamamagitan ng browser. Suriin lahat ito at tangkain muli ang paggamit ng pahinang ito.

Mag-scroll pababa sa pinaka-ibaba ng pahina at pindutin ang "Ituloy" na link.

Makakakita ka ng porma kung saan maitatakda mo ang marami pang baryabol para sa pagsasaayos ng instalasyon mo, tulad ng umiiral na wika SMTP host at iba pa. Huwag kang mag-alala kung hindi mo magawa ang lahat ng gusto mo sa ngayon - maaari ka namang bumalik dito at iedit ang iba pang hindi mo naiayos sa pamamagitan ng pang-admin na interpeys. Ang mga umiiral ay dinisenyo na maging kapakipakinabang at ligtas para sa karamihang site. Mag-scroll pababa at iklik ang "Isave ang pagbabago".

Kung (at tangi kung) hindi ka makaalis sa pahinang ito, at hindi ka makapagpatuloy, may problema ang server mo na ang tawag ko ay problemang "buggy referrer". Madali itong kumpunihin: patayin mo lamang ang "secureforms" na setting, pagkatapos ay tangkain mong magpatuloy muli.

Pagkatapos ay makakakita ka ng marami pang pahina na naglalathala ng maraming mensahe ng estado habang isinasaayos nila ang lahat ng teybol na kinakailangan ng iba't-ibang modyul ng Moodle. Tulad din ng nauna, dapat ay kulay berde ang mga ito.

Mag-scroll sa pinaka-ibaba ng pahina at pindutin ang "Ituloy" na link.

Ang susunod na pahina ay isang porma na pagtatakdaan mo ng mga parameter para sa Moodle site at sa pambungad na pahina, tulad ng pangalan, anyo, deskripsiyon at iba pa. Punan ang mga kahon nito (puwede mong itong balikan anumang oras para baguhin), tapos ay pindutin ang "Isave ang mga pagbabago".

Bilang panghuli, hihilingin kang lumikha ng isang top-level na administrador na user para mapasok ang mga pahina ng admin. Punan ang mga kinakailangang detalye ng pangalan mo, email atbp., tapos ay iklik ang "Isave ang mga pagbabago". Hindi lahat ng pitak ay kailangan, pero kapag may mahalagang pitak kang nakaligtaan, huhudyatan kang muli para sagutan ang mga ito.

Tiyakin na natatandaan mo ang bansag at kontrasenyas na pinili mo para sa akawnt ng administrador, dahil kailangan ito para mapasok ang pahinang pang-administrasyon sa mga susunod na pagkakataon.

(Kung dahil sa anumang bagay ay nahinto ang pagluluklok mo, o kung nagka-error na pansistema, na pumipigil sa iyong lumagda gamit ang admin akawnt, kadalasan ay makakalagda ka sa pamamagitan ng umiiral na bansag na "admin", na may kontrasenyas na "admin".)

Kapag nagtagumpay ka na, ibabalik ka sa tahanang pahina ng bago mong site! Pansinin mo ang mga link na pang-administrador sa may ibabang-kaliwa ng pahina (lumilitaw din ang mga aytem na ito sa hiwalay na pahinang pang-Admin) - nakikita mo lamang ang mga aytem na ito dahil nakalagda ka bilang taong admin. Lahat nang iba pang pamamahalang ng Moodle na kakailanganin mo ay maaari nang gawin sa pamamagitan ng menung ito, tulad ng:

  • paglikha at pagbura ng mga kurso
  • paglikha at pag-edit ng mga akawnt ng tao
  • pamamahala ng mga akawnt ng guro
  • pagbabago ng mga setting na laganap sa buong site tulad ng tema atbp

Pero hindi pa tapos ang pagluluklok mo! May isa pang napakahalagang bagay na kailangang gawin (tingnan ang sususunod na seksiyon hinggil sa cron).

 

6. Isaayos ang cron -- IMPORTANTE!

Kailangan ng pagmamanman sa ilang modyul ng Moodle upang maisakatuparan ang mga gawain nito. Halimbawa, kailangan tingnan ng Moodle ang mga talakayan upang mai-email nito ang mga kopya ng post sa mga taong sumali.

Ang iskrip na gumagawa ng lahat ng ito ay nasa direktoryong admin, at ang pangalan ay cron.php. Kaya lamang, hindi nito kayang tumakbo sa sarili nito, kaya kailangan mong magsaayos ng mekanismo na magpapatakbo ng iskrip na ito nang regular (eg tuwing ikalima o ikasampung minuto). Mistula ngayon itong "pintig-ng-puso" upang maisakatuparan ng iskrip ang mga gawain nito sa panahong itinatakda ng bawat modyul. Ang ganitong uri ng regular na mekanismo ay tinatawag na serbisyong cron .

Gusto kong bigyang pansin na hindi kailangang ang makinang nagpapatakbo ng Moodle ang makinang gagawa ng cron. Halimbawa, kung limitado ang serbisyong web hosting mo, at wala itong serbisyong cron, puwede mong patakbuhin ang cron sa isa pang server o sa kompyuter mo sa bahay. Ang mahalaga lamang ay mapatakbo ang cron.php nang regular.

Magaang lamang ang paggamit ng kompyuter ng script na ito, kaya ang 5 minuto ay makatwiran na, pero kung nag-aalala ka sa kinakain nitong rekurso ng server, maaari mong pahabain ang agwat ng pagtakbo sa mga 15 minuto o kahit 30 minuto. Mas mabuting huwag masyadong pahabain ang agwat na ito dahil ang pagbalam sa paglabas-ng-liham ay makapagpapabagal sa aktibidad ng kurso.

Una, subukin kung tumatakbo ang iskrip sa pamamagitan ng pagpapatakbo nito nang direkta sa pamamagitan ng inyong browser: 

http://halimbawa.com/moodle/admin/cron.php

Pagkatapos, kailangan mong magsaayos ng isang paraan ng pagpapatakbo ng iskrip nang awtomatiko at regular.

Sa sistemang Windows

Ang pinasimpleng paraan ay gamitin ang maliit na paketeng ito moodle-cron-for-windows.zip na magpapadali sa gawaing ito sa pamamagitan ng pagluklok ng isang maliit na serbisyong pang-Windows. Patakbuhin mo ito at kalimutan na!

Sa serbisyong web hosting

Maaaring may pahinang pangweb ang nakaweb na control panel ninyo para maisaayos ang prosesong cron na ito. Halimbawa, sa sistemang Cpanel, maghanap ng buton na may pangalang "Cron jobs". Doon mailalagay mo ang mga Unix command na nakalista sa ibaba.

Paggamit ng command line sa Unix

Maraming command line program na magagamit mo sa pagtawag sa pahina mula sa command line. Maaring hindi lahat nang ito ay nasa isang server.

Halimbawa, maaari mong gamitin ang isang Unix utility tulad ng 'wget': 

wget -q -O /dev/null http://halimbawa.com/moodle/admin/cron.php

Pansinin na sa halimbawang ito ang output ay itinatapon (sa /dev/null).

Ganito rin ang paraan kung gagamitin mo ang lynx: 

lynx -dump http://halimbawa.com/moodle/admin/cron.php > /dev/null

O kaya'y maaari kang gumamit ng standalone na bersiyon ng PHP, na kinompayl para tumakbo sa command line. Ang kaigihan ng ganitong paraan ay hindi mapupuno ang web server log mo ng paulit-ulit na paghiling sa cron.php. Ang kahinaan ng paraang ito ay kailangan mo ng command-line na bersiyon ng php. 

/opt/bin/php /web/moodle/admin/cron.php

Paggamit ng crontab program sa Unix

Ang ginagawa lang naman ng Cpanel ay maglagay ng interpeys na pangweb para sa isang kagamitang pang-Unix na tinatawag na crontab. Kung may command line ka, maaari mong isaayos ang crontab sa sarili mo sa pamamagitan ng command na ito: 

crontab -e

pagkatapos ay idagdag ang isa sa mga command sa itaas tulad ng:

*/5 * * * * wget -q -O /dev/null http://halimbawa.com/moodle/admin/cron.php

Kadalasan, ipapasa ka ng "crontab" command sa 'vi' editor. Makakapasok ka sa "insert mode" sa pamamagitan ng pagpindot ng "i", tapos ay itype mo ang linya sa itaas, tapos ay lumabas sa insert mode sa pamamagitan ng pagpindot ng ESC. Makapagse-save at makakalabas ka sa pamamagitan ng pagtype ng ":wq", o umayaw nang hindi nagse-save sa pamamagitan ng ":q!" (alisin ang mga panipi).

 

7. Lumikha ng bagong kurso

Ngayo't tumatakbo na nang maayos ang Moodle, bakit di ka lumikha ng bagong kurso na mapaglalaruan.

Piliin ang "Lumikha ng bagong kurso" sa pahinang Admin (o sa pang-admin na link sa tahanang pahina).

Punan ang porma, pag-ingatan mo ang pagtukoy sa anyo ng kurso. Hindi mo kailangang busisiin ang detalye sa panahong ito, dahil lahat ay maaaring baguhin mamaya ng mga guro. Tandaan na ang dilaw na ikon ng tulong ay nakakalat para magbigay ng nasa-kontekstong tulong sa anumang aspeto.

Pindutin ang "Isave ang mga pagbabago", at ipapasa ka sa isang bagong porma kung saan puwede kang magtakda ng guro sa kurso. Maaari mo lamang idagdag ang mga akawnt ng tao na mayroon na sa porma na ito - kung gusto mong lumikha ng bagong akawnt ng guro, hilingin mo ang guro na lumikha ng akawnt para sa sarili niya (tingnan ang pahinang panglagda), o lumikha ng isa para sa kanila sa pamamagitan ng "Magdagdag ng bagong user" sa pahinang pang-Admin.

Pag natapos na ito, puwede nang lagyan ng pasadyang katangian ang kurso, at magagawa ito sa pamamagitan ng "Mga Kurso" na link sa tahanang pahina.

Tingnan ang "Manwal ng Guro" para sa mga detalye ng paglikha ng kurso.
