Pag-aaplowd ng mga user

Una, tandaan na hindi naman talaga kailangang mag-angkat ng mga user nang lansakan - upang mabawasan ang gawain mo sa pagmementina, dapat ay pag-aralan mo muna ang ibang anyo ng pagsusuri ng pagkakakilanlan na hindi nangangailangan ng mano-manong pagsasaayos. Tulad ng pagkonekta sa yari nang panlabas na database o pagpapahintulot sa mga user na lumikha ng sarili nilang account. Tingnan ang seksiyon hinggil sa Pagtiyak sa Pagkakakilanlan sa mga pang-admin na menu.

Kung talagang nais mong mag-angkat ng maramihang user account mula sa isang text file, kakailanganin mong ipormat ang text file mo ng paganito:

  • Ang bawat linya sa file ay maglalaman ng isang rekord
  • Ang bawat rekord ay isang serye ng datos na pinaghihiwalay ng kuwit
  • Espesyal ang unang rekord ng file, at naglalaman ito ng listahan ng mga fieldname. Ito ang nagtatakda sa format ng kabuuang file.

    Mga kinakailangang fieldname: ang mga field na ito ay kailangang isama sa unang rekord, at bigyang kahulugan ayon sa bawat user

    username, password, firstname, lastname, email

    Mga default na fieldname: opsiyonal ang mga ito - kapag hindi ito isinama, ang mga halaga ay kukunin sa pangunahing admin

    institution, department, city, country, lang, auth, timezone

    Mga opsiyonal na fieldname: ganap na opsiyonal ang mga ito. Ang mga pangalan ng kurso ay ang "maiikling pangalan" ng mga kurso - kung mayroon nito, ang user ay ieenrol bilang mag-aaral sa mga kursong itinakda. Ang mga pangalan ng pangkat ay dapat kaugnay ng angkop na kurso, a.b. group1 sa course1, atbp.

    idnumber, icq, phone1, phone2, address, url, description, mailformat, maildisplay, htmleditor, autosubscribe, course1, course2, course3, course4, course5, group1, group2, group3, group4, group5, type1, type2, type3, type4, type5

  • Ang mga kuwit sa loob ng datos ay kailangang iencode na &#44 - awtomatikong idedecode ng iskrip ang mga ito pabalik sa kuwit.
  • Para sa mga Boolean na field, gumamit ng 0 para sa ditotoo at 1 para sa totoo.
  • Ang mga type ay ginagamit para sabihin sa Moodle kung ang user ay mag-aaral o guro, kung may kaugnay itong kurso (hal. type2 ay kaugnay ng course2). 1 = Mag-aaral, 2 = Nakapag-eedit na Guro, at 3 = Dinakapag-eedit na Guro. Kapag iniwang blangko ang type, o kung walang kursong itinakda, ang default na user ay mag-aaral.
  • Para sa mga kurso, gamiting ang maikling pangalan ng kurso
  • Tala: Kung ang user ay nakarehistro na sa Moodle user database, ihuhudyat ng script ang userid na bilang (database index) ng user na iyon, at ieenrol ang user bilang mag-aaral sa anumang itinakdang kurso NANG HINDI binabago ang iba pang itinakdang datos.

Halimbawa ng isang balidong file na pang-importa:

username, password, firstname, lastname, email, lang, idnumber, maildisplay, course1, group1, type1
jonest, verysecret, Tom, Jones, jonest@someplace.edu, en, 3663737, 1, Intro101, Section 1, 1
reznort, somesecret, Trent, Reznor, reznort@someplace.edu, en_us, 6736733, 0, Advanced202, Section 3, 3

Paggawang bago sa mga kasalukuyang account

Ang default na inaakala ng Moodle ay lilikha ka ng bagong user account, at nilalagpasan nito ang mga rekord na ang username ay tumutugma sa isang kasalukuyang account. Gayunpaman, kapag itinakda mo ang "Gawing bago ang mga kasalukuyang account" sa Oo, gagawing bago ang kasalukuyang user account.

Habang ginagawang bago ang mga kasalukuyang account, mababago mo rin ang mga username. Itakda ang "Pahintulutan ang pagbabago ng pangalan" sa Oo at isama sa file mo ang isang field na may pangalang oldusername.

Babala: ang anumang error sa paggawang bago ng mga kasalukuyang account ay magkakaroon ng masamang epekto sa mga user mo. Mag-ingat sa paggamit ng mga opsiyon sa paggawang bago.
