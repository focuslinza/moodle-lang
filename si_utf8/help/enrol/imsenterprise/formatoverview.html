හැඩතල සමාලෝචනය

පහත දැක්වෙන්නේ ආකෘතිකමය IMS Enterperice දත්ත ගොනුවක් සදහා සරල උපදෙස මාලාවකි. ඩැඩිවිස්තර සදහා IMS Enterprise අඩවි‍යට පිවිසෙන්න

ඔබට පේනු මෘදුකාංග(plugin) සකස් කලහැකි IMS දත්තාංග වල මොනව‍ දැයි අදුනාගත ගැනීමට අනුගත සාරංශය කියැවා බලන්න.

IMS Enterprise ගොනු හැඩතලය සදහා මූලික උපදෙස මාලාව

For any IMS-style enrolment you need a <group> tag which specifies the course, a <person> tag which specifies the user account, and a <membership> tag containing <member> tags which specify a person's role within a given course.

ඕනෑම IMS-හැඩයක් ලේඛන ගතකිරීමට ඔබට , පාඨමාලාව දැක්වෙන <group> ටැගයද භාවිතාකර‍න්නාගේ ගිණුම දැක්වෙන <person>ටැගය සහ <member> ටැගය , ඵහි පාඨමාලාව තුල දී භාවිතාකරන්නාගේ භූමිකාව සදහන් වන <membership> ටැගයන් ද අවශ්‍යය වේ.

මූඩ්ල් දත්තසමුච්චය සදහා යොදාගන්නා සංඛ්‍යාත්මක යතුරු interoperable දත්ත නොවේ- මූඩ්ල් දත්ත සමුච්චයට, නිමල් 20 වැනි භාවිතකරන්නා ඵකතු කල බව ශිෂ්‍යය දත්ත පද්ධතිය නොදනී- මේ නිසා මේවැනි යතුරු, හුවමාරු කලහැකි දත්ත වර්ගයක් නොවේ.

සාමන්‍යයෙන් පාඨමාලාවක් සදහා ‍යොමු ‍අංකයක් සහ නමක් ඇත, අපි යම් පාඨමාලාවක යොමු ‍අංකය: MOODLE101 යැයි සිතමු. දැන් ඔබ මේ පාඨමාලාව නිර්වචනය සදහා කේතය:

  <group>
    <sourcedid>
      <source>මාගේ-දත්ත-පද්ධතිය </source>
      <id>MOODLE101</id>
    </sourcedid>
    <description>
      <short>Moodle 101</short>
    </description>
  </group>

The enrolment script will look for a course with code MOODLE101, and (optionally) create it if it doesn't exist. Similarly for the person - let's assume it's "jmoodle":

ලේඛන ගතකිරීමේ විධානාවලිය(enrolment script) , MOODLE101 කේතය සහිත පාඨමාලාව ‍සොයයි. ඵය ‍නොමැතනම් ඵම කේතය නිර්මාණය කරයි(විකල්පයකි). උදාහරණයක් ලෙස "jmoodle" ගනිමු:

  <person>
    <sourcedid>
      <source>මාගේ-දත්ත-පද්ධතිය</source>
      <id>jmoodle</id>
    </sourcedid>
    <userid>jmoodle</userid>
    <name>
      <fn>නිමල් ගේ මූඩ්ල්</fn>
      <n>
        <family>MOODLE</family>
        <given>නිමල්</given>
      </n>
    </name>
  </person>

නිමල්ට ගිණුමක් නැති අවස්ථාවක, විධානාවලිය ඔහුට ගිණුමක් නිර්මාණය කරයි(විකල්පයකි).

Let's now look at the membership, adding the person to the course:

දැන් සාමාඡිකත්වය ගැන බලමු, පාඨමාලාවට පුද්ගලයෙකු ඵකතු කරමු:

    <sourcedid>

    </sourcedid>

IMS Enterprise වි‍ශේෂණය අනුව, ආරම්භක සහ අවසාන දිනය සදහන් කිරීමට පහසුකම් ඇත. ඵය <timeframe> ටැලයට අවශ්‍ය නම් ඇතුල් කල හැකිය.

පුද්ගලයෙකු පාඨමාලාවට ඵකතු දැනටමත් ඵකතු කල ඇතිවිට, විධානාවලිය සංස්කරණය අවශ්‍ය‍ය නැත. ඔවුන් කංඩායම් ගතකර නැතිනම්, සදහන් කල කංඩායම අඩංගු කරවයි.
