HTML 편집기의 사용

쪽을 편집할 때에 이 옵션은 일반 텍스트 편집기 대신 HTML 편집기(다음부터는 그냥 편집기라고 부름)를 사용할 수 있게 해 준다. HTML 편집기는 간단한 문서의 형태를 바꾸거나 연결고리와 같은 좀 더 "향상된" 기능들을 쉽게 사용할 수 있게 한다.

이 편집기를 사용하려면 다음에 있는 편집기 사용난을 클릭하면 된다. 편집기는 개개의 항목에 대해서만 적용된다. 예를 들어 반응 1에 대해 편집기 사용을 켜면, 반응 1에 대한 내용을 편집할 때만 적용되고 여타의 반응난은 영향받지 않는다.

만일 어떤 쪽의 항목을 편집기로 작성해야 할 필요가 있으면, 우선 편집기 사용난을 클릭한다.(체크표가 나타나야 한다) 그러면 편집기에서 편집할 수 있게 다시 표시된다. 편집이 다 끝나면 화면의 하단에 있는 쪽 다시보기를 눌러 편집을 마친다.

편집기를 사용할 때 유의해야할 사항이 몇 가지 있다. 대개의 경우에는 문제를 일으키지 않는다. 다만 주관식 단답형 문제와 수치형 문항을 작성할 때는 편집기 사용을 추천하지 않는다. 편집기는 별도의 "보이지 않는" HTML태그를 십입할 수 있기 때문에 비교작업을 멈추게 한다. 게다가 갈래표의 설명은 버튼으로 사용되기 때문에 만일 여기에 어떠한 HTML태그라도 포함하게 되면 문제를 일으킨다. 선다형 문항에서와 같이 단순히 표시하기 위해 사용하는 답안은 아무런 문제가 없다. 또한 피드백을 주기위한 내용 편집에 편집기를 쓰는데도 아무런 문제가 발생하지 않는다.

만일 문서를 입력하는데 문제를 일으킬 것 같다고 생각되면 편집기를 끄면(편집기 사용난에 기표 삭제)된다. 그러면 입력화면이 다시 표시되면서 문서 입력난에 "원래의" 문서를 볼 수 있고 원하는 대로 문서를 편집할 수 있다. 사실 특별한 항목에 대해 반복적으로 편집기를 바꾼다고 해서 문서 내용이 바뀌는 것은 아니다.

편집기 사용 "상태" 옵션은 각 항목과 함께 저장된다. 따라서 에디터를 사용했던 항목을 재 편집할 때는 편집기가 다시 나타난다.

마지막으로 브라우저의 측면에서 편집기로 작성한 문서는 상당히 자원을 잡아먹는 "무거운" 것이다. 따라서 편집하려는 내용이 꼭 편집기로 편집해야 할 내용이 아니라면, 표준 입력기를 사용하는 편이 빠르고 자원도 절약할 수 있다.
